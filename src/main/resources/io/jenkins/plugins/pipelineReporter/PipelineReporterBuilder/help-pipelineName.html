Specify the name of the pipeline project.

Supporting the following scenarios:

Pipeline Name Specified Build Number Specified Scenario
-------------------------------------
N N executed in the same pipeline, no need to define the pipeline and number.
code will fetch it from actual execution object (run)
Y N executed with a specific build number, maybe user
needs to to run on same build for different pipelines.
(usually this is needed for testing report look and feel...)
N Y executed with a specific pipeline job name and
without a build number - expected to get latest build of specific pipeline
Y Y executes with a specific pipeline job name and build number.
most common when a pipeline will be executed in a different job that and
pass the parameters as inputs, enabling to specify different build number then last one

There are no validation in design time that pipeline project exists, to support when pipeline is created using DSL.
When plugin is executed it validates the pipeline project exists.
